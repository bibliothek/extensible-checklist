  <what-built>
Complete checklist workflow system spanning:
- Checklist management API (list, delete, item operations)
- Checklist list page with progress indicators
- Checklist working page with full interaction capabilities
- All CHKL requirements (CHKL-01 through CHKL-08)
  </what-built>

  <how-to-verify>
**Prerequisites:**
1. Start development server: `npm run dev`
2. Log in to application
3. Ensure you have at least 2-3 templates created (with some overlapping items for merge testing)

**Test Sequence:**

**1. Checklist List View (CHKL-01, CHKL-08)**
- Visit http://localhost:3000/checklists
- Verify page loads without errors
- If no checklists: verify empty state shows with "Create from Templates" button
- Create 2-3 checklists via /checklists/new (select different template combinations)
- Return to /checklists
- Expected results:
  - [ ] All created checklists appear in list
  - [ ] Most recent checklist is at top (sorted by date)
  - [ ] Each shows name, progress (0/N items), progress bar, created date
  - [ ] Each has clickable name link and delete button
- Test delete:
  - [ ] Click delete button, confirm dialog appears
  - [ ] Confirm deletion, checklist removed from list
  - [ ] Page doesn't crash, remaining checklists still visible

**2. Open and Navigate Checklist (CHKL-02)**
- From /checklists, click a checklist name
- Expected results:
  - [ ] Navigates to /checklists/[id] page
  - [ ] Page loads without errors
  - [ ] Checklist name displayed as heading
  - [ ] Overall progress shown (e.g., "3 of 10 items completed")
  - [ ] Visual progress bar displays completion percentage
  - [ ] Back link to /checklists works
  - [ ] Items grouped by source template with headers
  - [ ] All items from instantiation are present

**3. Template Grouping and Collapse**
- On checklist detail page:
  - [ ] Items are grouped by source template (each group has header)
  - [ ] Groups are expanded by default, all items visible
  - [ ] Click collapse/expand toggle on a group
  - [ ] Group items hide when collapsed, show when expanded
  - [ ] Multiple groups can be collapsed independently
  - [ ] If checklist has "Custom" items (added after creation), they appear in separate group

**4. Complete Items (CHKL-03, CHKL-04)**
- Check off several items:
  - [ ] Click checkbox next to item
  - [ ] Item marked complete instantly (optimistic UI)
  - [ ] Progress updates immediately ("4 of 10 items completed")
  - [ ] Progress bar visually updates
  - [ ] Refresh page, completion state persists
- Uncheck items:
  - [ ] Click checked checkbox
  - [ ] Item marked incomplete instantly
  - [ ] Progress decreases accordingly
  - [ ] Changes persist after refresh

**5. Add New Items (CHKL-05)**
- Scroll to bottom of checklist:
  - [ ] "Add Item" form is present
  - [ ] Type text in input field
  - [ ] Press Enter key - item added, appears in list
  - [ ] Add another item using "Add Item" button instead of Enter
  - [ ] New item appears instantly (optimistic UI)
  - [ ] New items appear in "Custom" group or at bottom
  - [ ] Input field clears after successful add
  - [ ] Refresh page, new items persist

**6. Remove Items (CHKL-06)**
- Delete an item:
  - [ ] Click delete/X button next to an item
  - [ ] Confirmation dialog appears
  - [ ] Confirm deletion, item removed from list instantly
  - [ ] Progress updates if completed item was deleted
  - [ ] Cancel deletion works (item stays)
  - [ ] Refresh page, deleted item doesn't reappear

**7. Reorder Items (CHKL-07)**
- Test reordering within a group:
  - [ ] Up/down arrow buttons visible for each item
  - [ ] Click down arrow on first item - swaps with second
  - [ ] Click up arrow on second item - swaps back
  - [ ] Items reorder instantly (optimistic UI)
  - [ ] Refresh page, order persists
  - [ ] Reorder items at top/bottom of group (edge cases)

**8. Edit Item Text**
- Inline editing:
  - [ ] Click item text, cursor appears (editable)
  - [ ] Modify text (add/remove/change words)
  - [ ] Click outside item or tab away (blur)
  - [ ] Changes save automatically
  - [ ] Refresh page, edited text persists

**9. Responsive Design**
- Test mobile viewport:
  - [ ] Resize browser to mobile width (375px)
  - [ ] List page: cards stack vertically, remain readable
  - [ ] Detail page: progress bar, items, buttons all usable
  - [ ] Checkboxes and buttons are touch-friendly sized
  - [ ] No horizontal scroll
- Test tablet viewport:
  - [ ] Resize to tablet width (768px)
  - [ ] Layout adapts appropriately
  - [ ] All features remain accessible

**10. Dark Mode**
- Toggle system dark mode or use browser dev tools:
  - [ ] List page has dark mode styling
  - [ ] Detail page has dark mode styling
  - [ ] Text remains readable
  - [ ] Progress bars visible
  - [ ] Hover states work in dark mode

**11. Error Handling**
- Test error cases:
  - [ ] Visit /checklists/invalid-id - shows appropriate error
  - [ ] Turn off dev server mid-session - operations show error messages
  - [ ] Turn server back on - can recover and continue working

**12. Complete Workflow End-to-End**
- Full user journey:
  1. [ ] Start at /checklists (empty state)
  2. [ ] Click "Create from Templates"
  3. [ ] Select multiple templates, create checklist
  4. [ ] Redirected to list, new checklist appears
  5. [ ] Open checklist, work through items checking them off
  6. [ ] Add a few custom items
  7. [ ] Reorder some items
  8. [ ] Delete an item
  9. [ ] Complete all items (100% progress)
  10. [ ] Return to list, verify progress shows 100%
  11. [ ] Delete completed checklist
  12. [ ] Create new checklist for next use
